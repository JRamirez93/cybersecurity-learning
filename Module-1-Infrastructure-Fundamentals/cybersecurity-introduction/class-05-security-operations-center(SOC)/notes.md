# Class 05 – Security Operations Center (SOC)

Notes for **Class 05 – SOC** in Cybersecurity Introduction.  
Explains key SOC concepts in a clear, beginner-friendly way.

---

## 1. Vulnerability Assessment and Pentesting

- **Vulnerability Assessment** – Identifying weaknesses in systems before attackers exploit them.  
- **Pentesting (Penetration Testing / Red Teaming)** – Simulating attacks to find security gaps.  

## 2. Hardening Infrastructure

- Applying configurations and controls to reduce attack surfaces.  

## 3. Incident and Disaster Response (Blue Team)

- **Incident Response** – Procedures to detect, respond to, and recover from security incidents.  
- **Business Continuity Planning (BCP)** – Ensure critical operations continue during incidents.  
- **Disaster Recovery Planning (DRP)** – Procedures to recover systems after disasters.

---

## 4. Secure Development Lifecycle (SDLC)

- **SDLC** – Structured approach to software development.  
- Integrate security at each stage (requirements, design, coding, testing, deployment).  
- **DevOps** – Combines software development and operations; includes **DevSecOps** for security integration.

---

## 5. Team Roles in Security

- **Red Team** – Offensive team simulating attacks.  
- **Blue Team** – Defensive team protecting systems.  
- **Purple Team** – Collaboration between Red and Blue teams for continuous improvement.  
- **SOC (Security Operations Center)** – Centralized unit monitoring, detecting, and responding to security incidents.
