# Lab 02 – Phishing Site Navigation

## Description

In this lab, I practiced **identifying phishing websites** and understanding how browser filters prevent malicious activity.  
The objective was to explore PhishTank and other platforms to evaluate real-world phishing sites.

---

## Lab Walkthrough

### Step 1 – Access PhishTank

1. Navigate to [PhishTank.com](https://www.phishtank.com).  
2. Search for at least three websites listed as potential phishing sites.

![PhishTank Search](screenshots/lab2/Step1.png)

### Step 2 – Evaluate Automatic Filters

1. Visit each identified website using a web browser.  
2. Observe if the browser blocks access or warns about potential threats.  
3. Take notes on browser response.

![Browser Filter](screenshots/lab2/Step2.png)

### Step 3 – Analyze Content

- Check the URL, links, and page content.  
- Document any signs that indicate the site is impersonating a legitimate service.  
- Determine whether the site should be reported if missing from PhishTank’s database.

![Site Analysis](screenshots/lab2/Step3.png)

### Step 4 – Optional Reporting

1. If a site is not cataloged, submit it to PhishTank following the instructions for community contribution.

---

## Actions Performed

1. Conducted searches for phishing websites on PhishTank.  
2. Verified browser filtering mechanisms.  
3. Analyzed content and determined potential risk.  
4. Learned reporting process for new phishing sites.

---

## Tools Used

- Web browser (Chrome/Edge/Firefox)  
- PhishTank platform  

---

## Key Takeaways

- PhishTank provides a collaborative way to track and report phishing sites.  
- Browser security filters are an important first layer of defense.  
- Understanding phishing techniques helps anticipate and mitigate attacks.
