# Class 01 – Cybersecurity Fundamentals

## Overview

This class introduces the fundamental concepts of cybersecurity, including security principles, authentication mechanisms, governance, and regulatory considerations. The objective is to establish a conceptual foundation for understanding how organizations protect digital assets and manage security risks.

---

# Key Concepts

## Cybersecurity vs Information Security

Information Security focuses on protecting **information assets**, while Cybersecurity specifically focuses on protecting **systems, networks, and digital infrastructure from cyber threats**.

Cybersecurity is therefore considered a **subset of Information Security**.

---

## Core Areas of Information Security

Information security is typically divided into several domains:

* Network Security
* Application Security
* Endpoint Security
* Identity and Access Management (IAM)
* Governance, Risk and Compliance (GRC)
* Incident Response
* Security Operations

---

## CIA Triad

The **CIA Triad** is the foundational model used to guide cybersecurity policies.

* **Confidentiality** – Ensuring that information is accessible only to authorized users
* **Integrity** – Protecting information from unauthorized modification
* **Availability** – Ensuring systems and data are accessible when needed

These three principles guide most security frameworks and architectures.

---

## Authentication, Authorization and Accounting (AAA)

AAA is a framework used to control access to systems.

### Authentication

Authentication verifies the identity of a user.

Examples:

* Password
* Smart cards
* Biometrics
* Multi-Factor Authentication (MFA)

### Authorization

Authorization determines **what an authenticated user is allowed to do** within a system.

Examples:

* File access permissions
* Administrative privileges
* Application roles

### Accounting

Accounting tracks user activity for **auditing and monitoring purposes**.

Examples:

* Login logs
* Access logs
* Activity monitoring

---

## Identity Verification

Identity verification ensures that a user is truly who they claim to be.

Common methods include:

* Knowledge factors (passwords)
* Possession factors (tokens, cards)
* Inherence factors (biometrics)

---

## Password Security Best Practices

Strong password policies typically include:

* Minimum length requirements
* Complexity rules
* Expiration policies
* Password history restrictions
* Multi-factor authentication

---

## Principle of Least Privilege

The **Principle of Least Privilege (PoLP)** states that users should only have the minimum permissions necessary to perform their tasks.

Benefits:

* Reduces attack surface
* Limits damage from compromised accounts
* Improves system control

---

## Non-Repudiation

Non-repudiation ensures that a user **cannot deny performing an action**.

Common mechanisms include:

* Digital signatures
* Logging systems
* Cryptographic verification

---

## Implicit Deny

The **Implicit Deny principle** means that any access request that is not explicitly allowed should automatically be denied.

This is a fundamental concept in secure system design.

---

## Legal and Regulatory Considerations

Organizations must comply with various legal and regulatory frameworks related to cybersecurity, such as:

* Data protection regulations
* Privacy laws
* Security compliance standards

Compliance helps ensure that organizations protect sensitive information responsibly.

---

## Due Care and Due Diligence

### Due Care

Refers to implementing appropriate security controls to protect organizational assets.

### Due Diligence

Refers to the continuous process of **evaluating risks and ensuring controls remain effective**.

---

## Information Security Governance

Security governance defines **how an organization manages cybersecurity policies, procedures, and responsibilities**.

It ensures that:

* Security aligns with business objectives
* Risks are properly managed
* Compliance requirements are met
