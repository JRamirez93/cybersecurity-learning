# Integrative Cybersecurity Project

This directory contains the **final integrative assessment** developed during a cybersecurity fundamentals course.
The objective of this project is to apply the concepts learned throughout the course in practical scenarios involving system security, risk analysis, auditing, and social engineering.

The project is divided into five stages, each addressing a different area of cybersecurity practice.

---

# Project Stages

## Stage 1 — Windows User Account Security

Implementation of security measures for user accounts in a Windows 10 environment.

Tasks included:

* Creating user accounts with secure passwords
* Modifying Windows password policies
* Verifying password strength and account security

This stage focuses on **identity and access management fundamentals**.

---

## Stage 2 — Risk Management

A risk analysis scenario was developed for a fictional cybersecurity company.

Activities included:

* Identification of information assets
* Identification of potential threats
* Risk evaluation using probability and impact values
* Creation of a qualitative and quantitative risk matrix

This stage introduces **risk assessment methodologies used in cybersecurity management**.

---

## Stage 3 — Security Controls and Auditing

A case study based on the historical data breaches suffered by **Yahoo** in 2013 and 2014.

The analysis included:

* Physical security controls
* Logical access controls
* Access control models (MAC, DAC, RBAC, ABAC)
* Audit controls and monitoring
* Security audit frameworks

The objective was to understand how security controls and monitoring mechanisms can prevent or mitigate large-scale breaches.

---

## Stage 4 — Social Engineering Attack

A practical exercise using **Social-Engineer Toolkit** in **Kali Linux**.

The goal of this stage was to:

* Understand the mechanics of social engineering attacks
* Simulate an attack scenario
* Analyze why users may fall victim to manipulation techniques
* Reinforce the importance of security awareness training

---

## Stage 5 — Comprehensive Security Strategy with ISO 27001

The final stage focuses on designing a **comprehensive security strategy** aligned with **ISO/IEC 27001**.

Activities included:

* Security evaluation of a selected organization
* Passive reconnaissance and information gathering
* Identification of exposed information
* Vulnerability analysis
* Hardening recommendations
* Creation of a risk matrix
* Definition of applicable ISO 27001 security controls

This stage integrates both **Red Team and Blue Team perspectives** to develop a structured security approach.

---

# Skills Demonstrated

This project demonstrates practical understanding of:

* Windows account security
* Risk analysis and asset identification
* Security control implementation
* Security auditing and monitoring
* Social engineering techniques
* Security governance and compliance
* Information security frameworks

---

# Purpose of the Project

The purpose of this integrative assessment is to consolidate cybersecurity fundamentals through applied exercises that simulate real-world security challenges and organizational security analysis.
